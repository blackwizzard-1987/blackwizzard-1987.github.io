### Dify更换对象存储

因项目涉及部分内部文件不允许留痕，因此将Dify的对象存储和业务系统统一是最好的，方便管理。

Dify默认是将用户上传的文件存储在服务器上的docker目录里面，并在框架后台表中记录，具体如下：

```html
数据库表：upload_files
容器：docker-api-1
路径：/app/api/storage/upload_files/某id
docker cp docker-api-1:/app/api/storage/ + key（表中存的路径）+ path（目标路径）
```

切换为minio（兼容S3协议）操作如下：

```html
-- 修改.env文件
1、找到dify源码的docker目录下的.env文件
2、找到# File Storage Configuration 这个配置部分
3、修改STORAGE_TYPE，从opendal改为s3
4、找到# S3 Configuration
5、修改以下参数，配置为minio的参数
S3_ENDPOINT=http://ip:端口
S3_REGION=cn-xxx-1
S3_BUCKET_NAME=difyai
S3_ACCESS_KEY=xxx
S3_SECRET_KEY=xxx
注意：这里的桶一定要先创建好，属性为private（安全方面考虑）
6、修改后，将原本地目录下面的privkeys迁移到S3的根目录下面，如difyai/privkeys
7、重启docker
```

完成后，Dify平台上传的文件将直接进入minio设定好的桶+路径中。
